1 前言

本文主要介绍在后端接收前端VO数据时的参数合法性校验；涉及点：

  • SpringMVC
  • Bean Validation 1.0 JSR-303规范
  • hibernate-validator

日常业务开发中总免不了和前端进行数据交互，对于前端的VO数据要进行合法性校验，比如新增Person，Person的年龄属性总不能是-1吧？

其中hibernate-validator就是实现了Bean Validation 1.0 JSR-303规范的非常便利的校验库，并且SpringBoot的spring-boot-starter-web模块中默认引用了这个实现库。

3 分组校验

到这里解决方案还不够完善，原因：

  • 新增person，需要校验name属性，姓名不能为空；
  • 但是对于查询接口而言，name属性无需校验，可以为空

那么如何让其自动的在新增时校验name，而在查询时不校验name呢？此时就需要分组校验

原理很简单，将bean中的属性进行分组，校验bean时指定要校验的组即可。

但要注意，分组时需要有一个类可以依附，一般采用的方式是在当前的POJO类中写接口，注意这个接口没有任何业务意义，仅仅是用来对校验进行分组时的一个依附而已。
