线程池的一点总结

1 线程池类 ThreadPoolExecutor

1.1 新建线程池

JDK的线程池通过ThreadPoolExecutor 类的构造函数实现。

参数名 含义
int corePoolSize 线程池核心线程数，即使线程处于空闲，也不会回收
int maximumPoolSize 线程池线程的上限
long keepAliveTime
TimeUnit unit 与keepAliveTime参数一起，定义出一个时间x。若为非corePoolSize的线程处于空闲状态的时间超过x，则会被回收
BlockingQueue workQueue 指定线程在何种类型的队列中排队
ThreadFactory threadFactory 新线程的产生方式
RejectedExecutionHandler handler 拒绝策略

1.3 测试代码执行顺序

  • 首先启动线程池，此时核心线程为0，队列为0；添加2个线程后，核心线程works为2。

  • 添加3个线程后，队列为1，核心线程池的线程仍然为2

  • 添加5个线程后，核心线程为2，队列为3，此时队列满了;接下来添加线程，则会往线程池中添加，上限是maximumPoolSize，此处是4

  • 添加7个线程后，线程池中线程为4，已满；队列中为3，也满；再添加第8个线程，则根据拒绝策略抛出RejectedExecutionException 异常

2 静态方法便捷创建线程池

JDK默认提供Executors 工厂类，提供静态方法来便捷生成线程池，而不需要手动指定7个参数。

注意：不推荐使用静态方法创建线程池(阿里巴巴编程规约也明确指出这一点)。

2.6 JDK默认线程池总结

  1. FixedThreadPool 和 SingleThreadPool

允许的请求队列长度为Integer.MAX_VALUE，可能会堆积大量的请求线程，从而导致OOM。

  1. CachedThreadPool 和 ScheduledThreadPool

允许创建的线程数量为Integer.MAX_VALUE，可能会创建大量的线程，从而导致OOM

  1. WorkStealingPool使用Fork/Join框架，Fork/Join会将任务分发给线程池中的工作线程。

3 线程池拒绝策略

3.1 JDK默认拒绝策略

JDK默认实现了4种拒绝策略，各自对应策略和类图见下。线程池默认的拒绝行为是AbortPolicy

拒绝策略类名 行为
AbortPolicy 抛出RejectedExecutionException
DiscardPolicy 什么也不做,忽视掉超出的线程
DiscardOldestPolicy 丢弃执行队列中最老的任务，尝试为当前提交的任务腾出位置
CallerRunsPolicy 直接由提交任务者执行这个任务(线程直接调用run方法)

3.2 自定义拒绝策略

拒绝策略的行为是通过实现RejectedExecutionHandler 类，覆盖void rejectedExecution(Runnable r, ThreadPoolExecutor executor)方法来实现。在ThreadPoolExecutor 构造函数中进行传递即可

4 总结

  1. 如果任务量不大，可以使用无界线程池
  2. 如果要保证每一个请求都执行成功，应该看具体需求定制拒绝策略
