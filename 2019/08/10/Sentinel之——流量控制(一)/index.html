0 前言

本文先来整体实践一遍Sentinel对于流量控制的过程。

1 流量控制规则

监控应用流量的 QPS 或并发线程数等指标，当达到指定的阈值时对流量进行控制，以避免被瞬时的流量高峰冲垮。

Sentinel的流控分两大类：

  • 基于 QPS/并发数的流量控制
  • 基于调用关系的流量控制

名词含义：

  • 核心库：指的是sentinel-core包
  • 客户端：指的是引入了核心库的java应用程序
  • 控制台：即是Sentinel的Dashboard

本文使用 1.7.0-SNAPSHOT 版本来演示。

3 定义规则

在仅使用客户端，而没有引入控制台的时候，要定义资源规则，只能在客户端的程序中通过代码显示定义；如果客户端接入了控制台，那么可以在控制台通过界面进行资源的规则定义。

同一个资源可以创建多条限流规则。限流规则会被依次遍历，直到有规则触发限流或者所有规则遍历完毕。下面介绍如何通过代码显示定义规则。

流量控制规则的实体类是FlowRule，常用指标如下:

  • resource：资源名，即限流规则的作用对象
  • count: 限流阈值
  • grade: 限流阈值类型（QPS 或并发线程数）
  • limitApp: 流控针对的调用来源，若为 default 则不区分调用来源。默认为default
  • strategy: 调用关系限流策略。默认为不设置关系限流，直接根据设置的QPS或者线程数限流
  • controlBehavior: 流量控制效果(直接拒绝、Warm Up、匀速排队)，默认为直接拒绝，即是进入流控方法内

5 接入控制台

不接入控制台，仅使用核心库Sentinel是完全可以正常工作的。但是接入控制台会更方便。

接入控制台大体分两步：

  • 启动控制台应用，控制台作为独立的应用运行
  • 客户端启动时主动连接上控制台应用

5.1 启动控制台

可以从官方下载运行jar包下来，使用java -jar来启动控制台，控制台是使用SpringBoot开发的独立应用

这里为了调试方便，直接使用官方代码，在IDEA中启动sentinel-dashboard模块。

官方文档 中会带上一些-D参数启动，目的是把控制台同时也当成一个客户端，往自身连接并监控控制台自身的流控数据，这里不带任何参数直接启动。
